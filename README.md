# 프로그래밍 학습 프로젝트

이 저장소는 프로그래밍 기초부터 실전 프로젝트까지 단계적으로 학습한 내용을 정리한 것입니다. 각 프로젝트는 특정 문제를 해결하기 위한 목적을 가지고 있으며, 그 과정에서 얻은 지식과 경험을 기록했습니다.

---

## 📁 프로젝트 구조

```
code/
├── algorithm/          # 알고리즘 학습
├── data structure/     # 자료구조 문제 풀이
├── design pattern/     # 디자인 패턴 구현
├── DirectX/           # DirectX 그래픽 프로그래밍
├── GameMakerProject/  # GameMaker 게임 프로젝트
└── multi-threaded server/  # 멀티스레드 서버/클라이언트
```

---

## 🎯 프로젝트별 목적과 학습 내용

### 1. 알고리즘 기초 이해

#### 1.1 재귀 알고리즘의 원리 파악
**목적**: 재귀적 사고방식을 이해하고, 복잡한 문제를 작은 문제로 분해하여 해결하는 방법 학습

**프로젝트**: 피보나치 수열 & 하노이의 탑 (`code/algorithm/algo1/피보나치하노이.ipynb`)

**해결한 문제**:
- 피보나치 수열을 반복문으로 구현하여 시간 복잡도 개선
- 하노이의 탑 문제를 재귀적으로 분석하고 구현
- 하노이의 탑 이동 횟수 수식 도출: `2^n - 1`

**배운 점**:
- 재귀 함수의 호출 스택과 메모리 사용 이해
- 반복문 vs 재귀의 장단점 비교
- 문제를 재귀적으로 분해하는 사고 과정 체득

---

#### 1.2 자료구조 직접 구현 및 실전 활용
**목적**: 기본 자료구조를 직접 구현하여 내부 동작 원리를 이해하고, 실제 문제 해결에 적용

**프로젝트**: 연결 리스트, 스택, 큐 구현 (`code/algorithm/algo2/algo2/`)

**해결한 문제**:
- 더미 헤드를 활용한 연결 리스트로 삽입/삭제 연산 최적화
- 기존 `pop(index)` 함수를 `pop(index, k)`로 확장하여 연속 삭제 기능 구현
- 스택과 큐를 활용한 괄호 균형 검사 및 회문 검사 알고리즘

**배운 점**:
- 포인터/참조를 통한 메모리 관리 이해
- 재귀를 활용한 함수 확장 방법
- 자료구조 선택이 알고리즘 효율성에 미치는 영향 체감
- 스택과 큐의 LIFO/FIFO 특성을 활용한 문제 해결 패턴 학습

---

### 2. 자료구조 문제 해결 능력 향상

**목적**: 다양한 자료구조 문제를 C++로 해결하여 실전 코딩 능력 향상

**프로젝트**: 자료구조 문제 풀이 (`code/data structure/`)

**해결한 문제**:
- 재귀 함수를 활용한 수열 계산
- 다양한 자료구조를 조합한 복합 문제 해결

**배운 점**:
- C++ 문법과 메모리 관리
- 문제 분석 및 적절한 자료구조 선택 능력
- 알고리즘 설계 및 최적화 사고

---

### 3. 디자인 패턴을 통한 코드 구조 개선

**목적**: 유지보수 가능하고 확장성 있는 코드 구조 설계 방법 학습

#### 3.1 옵저버 패턴으로 결합도 낮추기
**프로젝트**: Unity 게임의 상태 관리 시스템 (`code/design pattern/디자인패턴/5. 옵저버 패턴 코드들.txt`)

**해결한 문제**:
- 게임 내 스태미너/체력 변경 시 UI를 수동으로 업데이트하던 문제
- 상태 변경 로직과 UI 표시 로직이 강하게 결합되어 있던 구조

**해결 방법**:
- Subject-Observer 패턴 적용
- 상태 변경 시 자동으로 모든 옵저버에게 알림
- UI 컴포넌트를 옵저버로 등록하여 자동 업데이트

**배운 점**:
- 디자인 패턴이 실제로 코드 품질을 개선하는 과정 체험
- 느슨한 결합(Loose Coupling)의 중요성 이해
- 이벤트 기반 프로그래밍의 장점 학습

#### 3.2 스테이트 패턴으로 상태 관리 체계화
**프로젝트**: 게임 상태 관리 (`code/design pattern/디자인패턴/7.스테이트 패턴 수정본.txt`)

**해결한 문제**:
- if-else 문으로 복잡하게 얽힌 상태 전환 로직
- 상태 추가 시 기존 코드 수정이 필요한 구조

**배운 점**:
- 상태를 객체로 캡슐화하여 관리하는 방법
- 상태 전환 로직의 명확화
- 확장 가능한 상태 관리 구조 설계

---

### 4. 저수준 그래픽 API 이해

**목적**: 게임 엔진의 내부 동작 원리를 이해하기 위해 그래픽 파이프라인을 직접 구현

**프로젝트**: DirectX 기반 3D 렌더링 엔진 (`code/DirectX/그래픽2/`)

**해결한 문제**:
- 3D 그래픽이 화면에 렌더링되는 전체 과정 이해
- 카메라 변환, 모델 변환, 뷰포트 변환 등 좌표계 변환 학습

**구현 내용**:
- DirectX 디바이스 초기화 및 관리
- 3D 모델 렌더링 파이프라인 구축
- 쉐이더를 통한 렌더링 제어
- 카메라 시스템 구현

**배운 점**:
- 그래픽 파이프라인의 각 단계 이해 (정점 처리 → 래스터화 → 픽셀 처리)
- 행렬 변환을 통한 3D 좌표계 변환 이해
- 게임 엔진이 내부적으로 어떻게 동작하는지 깊이 이해
- 저수준 API를 다루는 경험

---

### 5. 게임 개발 전 과정 경험

**목적**: 게임 개발의 전체 프로세스를 경험하고, 여러 시스템을 통합하여 완성도 있는 게임 제작

**프로젝트**: GameMaker 게임 프로젝트 (`code/GameMakerProject/Project1Aa.gmx/`)

**해결한 문제**:
- 캐릭터 이동, 공격, 적 AI 등 다양한 게임 시스템 구현
- 여러 시스템 간 상호작용 설계
- 게임의 완성도를 높이는 세부 기능 추가

**구현 내용**:
- 캐릭터 이동 및 공격 시스템
- 적 AI 및 전투 시스템
- 코인 수집 및 경제 시스템
- HP/스태미너 UI 시스템
- 다중 방(Room) 구성
- 사운드 효과 통합

**배운 점**:
- 게임 개발의 전체 라이프사이클 경험
- 여러 시스템을 통합하는 설계 능력
- 사용자 경험(UX)을 고려한 기능 설계
- 프로젝트 관리 및 단계별 개발 방법론

---

### 6. 네트워크 프로그래밍과 동시성 처리

**목적**: 멀티스레드 환경에서 여러 클라이언트를 동시에 처리하는 서버 구현

**프로젝트**: TCP 멀티스레드 서버/클라이언트 (`code/multi-threaded server/`)

**해결한 문제**:
- 단일 스레드 서버의 동시 접속 제한 문제
- 여러 클라이언트가 공유 데이터에 접근할 때의 동기화 문제
- 네트워크 통신의 신뢰성 보장

**해결 방법**:
- 각 클라이언트마다 별도 스레드 생성
- 공유 변수를 통한 데이터 동기화
- TCP 프로토콜을 활용한 신뢰성 있는 통신
- 에러 처리 및 연결 관리

**배운 점**:
- 소켓 프로그래밍의 기본 원리 이해
- 멀티스레드 프로그래밍과 동시성 처리
- 네트워크 통신 프로토콜 이해
- 서버-클라이언트 아키텍처 설계
- 동시성 문제와 해결 방법 학습

## 💡 핵심 학습 성과

이 프로젝트들을 통해 다음을 달성했습니다:

1. **알고리즘 사고력**: 재귀적 사고와 문제 분해 능력
2. **자료구조 이해**: 내부 동작 원리를 통한 올바른 선택 능력
3. **설계 능력**: 디자인 패턴을 활용한 확장 가능한 코드 구조 설계
4. **시스템 이해**: 저수준 API를 통한 시스템 내부 동작 이해
5. **통합 능력**: 여러 시스템을 통합하여 완성도 있는 프로젝트 제작
6. **동시성 처리**: 멀티스레드 환경에서의 프로그래밍 능력

각 프로젝트는 단순히 코드를 작성하는 것을 넘어, **왜 그렇게 설계해야 하는지**, **어떤 문제를 해결하는지**를 명확히 이해하는 것을 목표로 진행했습니다.
